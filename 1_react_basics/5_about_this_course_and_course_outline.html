Course Outline:
Basics & Foundation (Introducing key features)
  • Components & building UIs
  • Working with Events & Data: "props" and "state"
    • PROPS:
      • props are used to pass data.
      • Props can only be passed from parent component to child (unidirectional flow) Modifying state should happen with the setState ( ) method.
    • STATE:
      • state is for managing data.
      • State data can be modified by its own component, but is private (cannot be accessed from outside).
  • Styling react apps & components
  • Introduction into "react hooks"
    • Hooks are functions that let you “hook into” React state and lifecycle features from function components.
    • Hooks don't work inside classes — they let you use React without classes.
    • You can also create your own Hooks to reuse stateful behavior between different components.
Advanced Concepts (Building for Production)
  • Side effects, "Refs" & More React Hooks
    • Side effects
      • Side effects include: Modifying any external variable or object property (e.g., a global variable, or a variable in the parent function scope chain) Logging to the console.
    • "Refs"
      • The ref is used to return a reference to the element.
      • Refs should be avoided in most cases, however, they can be useful when we need DOM measurements or to add.
  • React's Context API & Redux
    • Context API
      • The React Context API is a way for a React app to effectively produce global variables that can be passed around.
    • Redux
      • Redux is a predictable state container designed to help you write JavaScript apps that behave consistently across client, server, and native environments and are easy to test.
      • While it's mostly used as a state management tool with React, you can use it with any other JavaScript framework or library.
  • Forms, HTTP requests & "Custom Hooks"
  • Routing, Deployment, NextJS & More
